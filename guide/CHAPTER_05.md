# 第5章：細部を語る（詳細ページと動的ルーティング）

一覧画面ができたら、次は一つ一つの資料を詳しく見るための「専用の部屋（詳細ページ）」が必要です。
しかし、資料が100個あれば100個ページを作るのでしょうか？……いいえ、プログラミングでは **「動的ルーティング」** という魔法を使い、1つのテンプレートで全ての資料を表示させます。

---

## 📂 フォルダ名にある `[displayId]` の意味

`src/app/document/[displayId]/page.tsx` という不思議なフォルダ名を見てください。
この `[ ]` で囲まれた部分は、**「どんな文字が入ってもいいですよ（変数）」** という意味です。

*   `.../document/DOC134-001` と打てば、`displayId` は "DOC134-001" になります。
*   `.../document/DOC134-999` と打てば、`displayId` は "DOC134-999" になります。

一つのファイルが、URLに合わせて中身を書き換えてくれるのです。

---

## 📝 詳細ページの全行解説 (src/app/document/[displayId]/page.tsx)

### 160-176行目：入り口（メインの関数）
まずはここが最初に実行されます。
```tsx
160: export default async function DocumentDetailPage({ // 160: 詳細ページを作る関数
161:   params,                                           // 161: URLの [displayId] の中身を受け取る
163: }: {
164:   params: Promise<{ displayId: string }>;
165: }) {
166:   const { displayId } = await params;               // 166: URLからID（例: DOC134-001）を取り出す
167:   const doc = await getDocumentByDisplayId(displayId); // 167: 引き出しから、そのIDの資料だけを取り出す
169:   if (!doc) {                                       // 169-171: もし資料がなかったら「見つかりません(404)」と出す
170:     notFound();
171:   }
174:   if (displayId === 'DOC134-000') {                 // 174-176: もし「零号」だったら、専用の豪華な画面を表示する
175:     return <DocumentZero doc={doc} />;
176:   }
```

### 184-230行目：通常の資料の表示
零号以外の、一般的な資料を表示する部分です。
```tsx
186: <header className="...">                           // 186: タイトルやIDを表示するヘッダーエリア
196:   <h1 className="text-4xl ...">{doc.title}</h1>    // 196: 資料名を大きく表示
202:   <p className="text-2xl ...">{doc.displayId}</p>  // 202: 識別番号を表示
211: {doc.thumbnailUrl && (                             // 211-220: 写真があれば、枠を付けて表示する
228: <ReactMarkdown>{doc.content}</ReactMarkdown>      // 228: マークダウン（文章）を綺麗に変換して表示
```

### 31-158行目：零号専用の豪華な演出（DocumentZero）
プログラミングができるようになると、特定の条件（このプロジェクトではID:000）の時だけ、**完全に別のデザイン**に切り替えることも可能です。
*   `mix-blend-multiply`: 背景と画像を馴染ませて「古い紙」のような質感にする。
*   `italic`: 管理者の手記であることを示すために斜体にする。
*   `iframe`: YouTubeなどの動画を埋め込んで表示する。

---

## 🎨 デザインのこだわり：マージン（余白）とパディング

このページでは `mb-16` (下の余白) や `pt-24` (上の余白) が多用されています。
初心者はついつい要素を詰め込みがちですが、**「余白は情報の価値を高める」** というのがこのアーカイブの哲学です。
何もない空間を作ることで、中心にある「資料」に視線が集中するように設計されています。
